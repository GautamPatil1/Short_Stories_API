Short Stories API Documentation

The Short Stories API is a robust and versatile API that provides developers access to a vast collection of captivating short stories across multiple genres. This API is designed to offer an immersive storytelling experience, allowing users to discover and explore intriguing narratives from the realms of fantasy, horror, mystery, and philosophy.
The Short Stories API is built using Django, a powerful web framework for building web applications, and the Django REST Framework (DRF), an extension to Django for creating RESTful APIs. It provides developers with a simple and convenient way to access a collection of short stories in various genres.

Architecture

The API follows a client-server architecture, where clients can make HTTP requests to the API server to retrieve short stories or perform other actions. The server processes the requests and sends back the corresponding responses in JSON format.

Base URL

The base URL for all API endpoints is: https://shortstoriesapi.gautampatil11.repl.co/

Endpoints

Random Story

Experience the element of surprise and serendipity by retrieving a captivating short story randomly selected from the vast repository of tales available.

/random/: This endpoint retrieves a random short story from the collection of stories. Each time a request is made to this endpoint, a different story is returned, allowing users to discover new stories with every request.

Endpoint: /random/

HTTP Method: GET

Response Format: JSON

Response Fields:

  • Name (string): The title of the randomly selected short story.
  • Author (string): The name of the talented author who crafted the story.
  • Cover (string): A visually captivating cover image that sets the mood for the story.
  • Genre (string): The overarching genre of the short story.
  • Sub-Genre (string): A more specific sub-genre classification that defines the story’s unique attributes and themes.
  • Content (string): The immersive text content of the short story that will transport readers into a world of imagination and emotion.

GET /random/

Story by Genre

Delve into the enchanting world of a specific genre by retrieving a captivating short story tailored to your preferences.

/<genre>/: These endpoints are used to retrieve a short story from a specific genre, such as fantasy, horror, mystery, or philosophy. By specifying the desired genre in the URL, users can access a story that aligns with their preferences.

Endpoint: /<genre>/

HTTP Method: GET

Response Format: JSON

Response Fields:

  • Genre (string): The broad genre category to which the short story belongs.
  • Name (string): The captivating title of the short story that encapsulates its essence.
  • Author (string): The masterful author responsible for crafting the story’s narrative.
  • Cover (string): An aesthetically pleasing cover image that visually represents the story’s genre and evokes anticipation.
  • Content (string): The immersive text content of the short story that vividly captures the imagination and emotions of readers.

  4. Philosophy

Multiple Stories by Genre

Immerse yourself in a collection of captivating short stories by retrieving multiple tales from a specific genre.

/<genre>/<count>/: These endpoints allow users to retrieve multiple short stories from a specific genre. Users can specify the desired number of stories they want to retrieve, making it convenient to access a batch of stories in one request.

Endpoint: <genre>/<count>/

HTTP Method: GET

Response Format: JSON

Response Fields:

  • Genre (string): The broad genre category to which the short stories belong.
  • Stories (array): An array of short stories, each containing the following fields:
    • Name (string): The captivating title of the short story.

Error Handling

The Short Stories API includes robust error handling mechanisms to ensure a smooth and error-free experience. The API provides informative error responses for various scenarios:

  • If a requested genre is not found, the API responds with a 404 Not Found status code and an error message indicating the invalid genre selection.
  • If no stories are found for a requested genre, the API responds with a 404 Not Found status code and an error message indicating that no stories are available for the selected genre.
  • If the count value provided for multiple stories is invalid (e.g., not a positive integer), the API responds with a 400 Bad Request status code and an error message indicating the invalid count value.
  • If the count value provided exceeds the available number of stories in a genre, the API responds with a 400 Bad Request status code and an error message indicating that there are not enough stories available for the requested count.

Please refer to the respective error response messages in the previous examples for more detailed explanations.
